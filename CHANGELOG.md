## [Unreleased]

### Changed

- 2026-07-13: Made the character RON assets (`assets/characters/*.character.ron`) the single source of truth for stats — removed all serde defaults from `CombatStatsDef` / `CharacterDef` so a template omitting any field now fails to deserialize rather than silently falling back, and added a data-driven `damage_variance` (`DamageVarianceDef { min, max }`) that flows from RON into the `DamageVariance` component at both spawn sites (`spawn_player` and the `spawn_enemies` `bsn!` scene). Negative deserialization tests lock in the no-defaults contract; `just ci` green (#38)
- 2026-06-20: Replaced the `bevy-inspector-egui` community debug inspector with Bevy's official diagnostics overlay — `src/debug/mod.rs` now wires `FpsOverlayPlugin` (from `bevy_dev_tools`), an on-screen FPS / frame-time readout toggled in-game with F12, rather than the egui right-click-to-inspect panel that had no Bevy 0.19-compatible release. The `debug-inspector` feature is renamed `debug-overlay` (enabling `bevy/bevy_dev_tools` instead of the removed `dep:bevy-inspector-egui` + `bevy/track_location`); `just run-debug` runs the game with the overlay again instead of being a no-op. Because `debug-overlay` is now a declared cargo feature, the `[lints.rust] unexpected_cfgs` workaround is dropped. The plugin stays a no-op without a `RenderApp` so headless `--features debug-overlay` tests stay green; default/release/wasm builds and tests never link `bevy_dev_tools`
- 2026-06-18: Migrated from Bevy 0.18.1 to Bevy 0.19 — `TextFont.font_size` is now a `FontSize` (every site wrapped in `FontSize::Px(...)`), and the deprecated run-condition combinator `SystemCondition::and` was replaced with `.and_then(...)` (same lazy short-circuit). The `debug-inspector` feature and its `bevy-inspector-egui` dependency are temporarily commented out because no Bevy 0.19-compatible release of that crate exists yet (it targets 0.18 on crates.io and git `main`); the inspector code and its `#[cfg(feature = "debug-inspector")]` gates are left in place (with the cfg declared via `[lints.rust]` so default builds stay warning-free) so the feature can be restored in one step once a compatible release ships — `just run-debug` is a no-op until then. The `bevy = "0.19"` manifest pin targets the crates.io release; until 0.19.0 is published there, regenerate `Cargo.lock` and build once it lands
- 2026-06-14: Debug inspector reworked from a whole-world dump to right-click-to-inspect — the `DebugPlugin` drops `WorldInspectorPlugin`, the `InspectorEnabled` resource, and the F12 toggle in favour of an `InspectedEntity` resource set by right-clicking (or Control+left-clicking) any sprite; an egui panel shows just that entity's components via `bevy_inspector::ui_for_entity` (sticky to the last-clicked entity, dismissed with Esc or the window ×, self-clearing when the selection despawns), alongside an Enemies list window for target-picking and a "jump to code" enabled by scoping `bevy/track_location` to the `debug-inspector` feature; `#[derive(Reflect)]` + `register_type` extended to the remaining battle components (`Player`, `Enemy`, `DisplayName`, `Defending`, `Targeted`, `EnemyHealthBar`) so every component expands in the panel, and a new `sync_enemy_label_text` system makes inspector `DisplayName` edits update world-space enemy labels live; `DebugPlugin` stays a no-op without a `RenderApp` so headless `--all-features` tests stay green (#22)
- 2026-06-13: Enabled `clippy::pedantic` with documented Bevy-friendly allow-backs in `Cargo.toml`; `just ci` green with no source-level violations (#10)

### Added

- 2026-06-17: WebAssembly browser build + Access-gated Cloudflare Pages deploy — the project now compiles to `wasm32-unknown-unknown` and the latest `main` is auto-deployed to Cloudflare Pages, gated by Cloudflare Access (Zero Trust) email login restricted to an allow-list. A `[target.'cfg(target_arch = "wasm32")']` block in `Cargo.toml` adds Bevy's `webgl2` backend and the `getrandom` `wasm_js` feature, paired with a `getrandom_backend="wasm_js"` cfg in `.cargo/config.toml` (the entropy source for `rand` in the browser, since `battle.seed` filesystem pinning is desktop-only). A Trunk entry point (`index.html` + `Trunk.toml`) bundles the wasm and copies `assets/`, with `just run-web`/`just build-web` recipes. A `deploy-cloudflare` GitHub Actions workflow (configured via `wrangler.toml`) builds the release bundle and deploys via Wrangler on every push to `main`; access control is enforced at Cloudflare's edge by an Access policy (One-time PIN email login) set up in the Zero Trust dashboard, so unauthenticated visitors never receive the game files. A dedicated size-optimized `wasm-release` cargo profile (`opt-level = "s"`, fat LTO, single codegen unit, `panic = "abort"`) paired with `wasm-opt -Oz` keeps the download small (~22 MB, ~7 MB gzipped) while leaving the native `release` profile at `opt-level = 3` for runtime speed
- 2026-06-16: Map exploration, random battles, and a game-over loop — the top-level `GameState` gains `Map` and `GameOver` (alongside `MainMenu`/`InBattle`), turning the single battle into a loop. New Game now resets progress and drops the player onto a generic-but-functional overworld (`map.rs` `MapPlugin`: a checkerboard background, a WASD/arrow-driven avatar clamped to the play area, and an HP readout), where accumulated walking distance triggers a random encounter into `InBattle`. A new `PlayerProgress` resource (seeded from the hero template, written back from the surviving `Health` on victory) carries hit points across fights, so winning returns to the map with damage persisted; losing routes to a `GameOver` screen (`game_over.rs`, mirroring the main menu's cursor/navigation) offering "Restart Game" (reset to full HP back on the map) and "Return to Title Screen". Battles are now re-enterable: the combatants, action menu, HUD/log, and selection indicator are tagged `DespawnOnExit(InBattle)` and re-spawned per encounter (replacing the global `run_once` spawn gate with a "no combatants present" gate), `TurnPhase` is rewound `OnEnter(InBattle)`, and an Enter-to-continue prompt on `BattleOver` hands the player off to the map (win) or game-over screen (loss). Headless tests cover avatar movement + encounter triggering (`tests/map_exploration.rs`), the victory→map HP persistence and defeat→game-over hand-off (`tests/battle_outcome.rs`), and the game-over menu dispatch (`tests/game_over.rs`)
- 2026-06-16: Start-up main menu — a top-level `GameState { MainMenu, InBattle }` (above the battle's own `TurnPhase`) with a `main_menu.rs` `MainMenuPlugin` rendering a centred "Bevy 2D RPG" title over New Game / Options / Credits rows, a yellow `>` cursor, and Up/Down (wrap) + Enter keyboard nav reusing the action menu's pure `cycle_index`; New Game switches to `InBattle` (the battle's combatants, action menu, HUD/log, and selection indicator now spawn `OnEnter(InBattle)` rather than at startup, and the four `BattleSet`s are gated on `in_state(InBattle)` so the menu and battle never double-read the keyboard), while Options and Credits are intentionally non-functional (log "not yet implemented" and stay on the menu); the 2D camera moved to a shared `GamePlugin` startup system so it outlives the menu→battle transition; a headless `tests/main_menu.rs` covers structure, wrap-around navigation, New Game→`InBattle` + menu tear-down, and the non-functional options, and `tests/smoke.rs` now drives the menu→battle transition
- 2026-06-14: Bevy "fast compiles" build optimizations from the [setup guide](https://bevy.org/learn/quick-start/getting-started/setup/) — a `.cargo/config.toml` mirroring Bevy's upstream `config_fast_builds.toml` (LLD already the Linux default; mold + nightly `share-generics`/`threads`/`no-embed-metadata` documented and ready to uncomment), a `dynamic_linking` cargo feature (`bevy/dynamic_linking`) with a `just run-fast` recipe for the fastest iterative dev builds, and explanatory comments on the existing dev `opt-level` profiles
- 2026-06-13: Debug inspector + polish (final milestone) — a `DebugPlugin` behind the `debug-inspector` cargo feature wiring `EguiPlugin::default()` + `WorldInspectorPlugin` (bevy-inspector-egui 0.36) gated on an F12-toggled `InspectorEnabled` resource via `run_if` (and a no-op when no `RenderApp` is present, so headless `--all-features` tests stay green); `#[derive(Reflect)]` + `register_type` on `BattleLayout`, `UiConfig`, `Health`, `CombatStats`, and `DamageVariance` so every tunable knob is live-editable; README run-debug/F12 docs, and a finalized `tests/smoke.rs` driving the full `GamePlugin` stack headless on a pinned seed (player + 1..=`MAX_ENEMIES` enemies spawn, 10 frames run without panic); `cargo build` without the feature proves egui compiles out (#8)
- 2026-06-13: HUD + battle log UI parity — a `BattleUiPlugin` driving the player name + percentage-width HP fill (with a "(defeated)" suffix on death) off `Changed<Health>`, dynamic alive-enemy name labels (rebuilt on death, yellow highlight following the `Targeted` marker), world-space enemy mini HP bars (track + fill scaled by health fraction under each sprite), and a battle-log panel spawning a `Text` child per `LogMessage`; a `UiConfig { action_menu_half_width, battle_log_half_width }` resource swapping the centre panel 200 px ↔ 350 px (menu shown in `PlayerTurn`/`Targeting`, log shown during `EnemyTurn`/`BattleOver`, log cleared `OnEnter(PlayerTurn)`); headless tests cover fill percent, defeated suffix, label-count drop, targeting highlight, log append + width swap, and a live `UiConfig` edit (#7)
- 2026-06-13: Enemy turn, Defend resolution, and game over — an `EnemyTurnQueue { pending: VecDeque<Entity>, timer: Timer }` built `OnEnter(EnemyTurn)` from alive enemies in index order and a `tick_enemy_turn` releaser (first attack immediate, then 1.0 s gaps, empty-queue hand-back to `PlayerTurn`) writing one `AttackRequested` per pop; `apply_attacks` halving the attacker's *attack value* before the formula while the target carries `Defending` (cleared `OnEnter(PlayerTurn)`, so it lasts exactly one enemy turn); `check_battle_end` extended with a player-death "Game Over!" branch (`BattleOver`, queue cleared so a mid-queue death stops remaining attacks) and a queryable `BattleResult { victory }` resource recording the outcome at each ending; deterministic headless tests via `TimeUpdateStrategy::ManualDuration` and a full-loop `tests/battle_flow.rs` exercising the whole battle orchestration (#6)
- 2026-06-13: Targeting, player attack, and victory — a `Targeting`-phase cursor cycling alive enemies (Left/Right wrap, Escape cancels, Enter confirms) with a `Targeted` yellow sprite tint and a `Mesh2d(Triangle2d)` selection indicator; `AttackRequested`/`DamageDealt` messages and a `Died` `EntityEvent` whose observer hides the defeated sprite; `combat::apply_attacks` (variance from `DamageRng`, `Health` mutation, "attacks/defeated" log lines) and `check_battle_end` ("Victory!" → `BattleOver`, else `EnemyTurn`); `Pickable` enemies with a per-entity `On<Pointer<Click>>` observer delegating to a pure `try_select_target` (`SpritePickingMode::BoundingBox`); headless tests cover the targeting cases (#5)
- 2026-06-13: Turn states + action menu — `TurnPhase` state machine and chained `BattleSet { Input, Resolve, Cleanup, Ui }` system sets; a bottom-anchored Fight/Items/Defend/Flee action menu with a visibility-toggled yellow `>` cursor and yellow/white highlight; player-turn-gated Up/Down (wrap) + Enter keyboard nav; Fight→Targeting, Items/Flee→log + EnemyTurn, Defend→`Defending` marker + queued message → EnemyTurn; a frame-buffered `LogMessage`; `Defending` cleared `OnEnter(PlayerTurn)`; headless tests cover the action-menu cases (#4)
- 2026-06-13: Character RON assets + battle spawning — `CharacterDef` as a loadable `Asset` with a `*.character.ron` `AssetLoader`, `hero`/`goblin` templates, and a seeded spawn that rolls 1–4 enemies, suffixes duplicate names, and lays out the player + enemy row from `BattleLayout` with a `Camera2d`; headless spawn tests (#3)
- 2026-06-13: Core domain logic — character components/definitions (serde-deserialized stat definitions), `parse_seed`/`read_seed_file`, `SpawnRng`/`DamageRng` (`ChaCha8Rng`), pure `compute_damage` (rounds to nearest), and `suffix_duplicate_names`, with unit tests (#2)
